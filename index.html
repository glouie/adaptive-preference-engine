Three types: selector (parent), variant (option), toggle (on/off)

🔗 Asymmetrical Associations

table ─────────────────→ data_structure_clarity  (strength: 0.95)
     ←──────────────────                           (strength: 0.50)

Why asymmetrical?
  table → data_structure: STRONG  (tables explain structure well)
  data_structure → table: WEAKER  (structure doesn't always need tables)

📚 Context Stack

Conversation level (highest priority)
Overrides everything for this chat session
Project level
Habits specific to a codebase or project
Base level (always active)
Your global defaults across all contexts

⚗️ Strength Formula

strength = frequency × trend × emotion × recency

Where:
  frequency  = how often used            (0 → 1)
  trend      = direction of change       (0.70 → 1.15)
  emotion    = user satisfaction         (0.50 → 1.00)
  recency    = time decay                (2% per day unused)

📡 How It Learns
Three signal types drive all learning in the engine
🔴 Correction Signal (Strongest)

When the agent proposes X and you want Y. This is the primary learning moment — the system updates association strengths, increments confidence for Y, decrements for X, and records the context.

🟢 Feedback Signal

Positive or negative emotional responses like "Yes, perfect!" or "That's not what I wanted." Emotional tone is extracted and used as a multiplier on confidence scores.

🔵 Usage Signal (Implicit)

What you choose over time, without explicit correction. Frequency of use contributes to the strength formula passively.

📖 Learning Example — Day by Day

🗓️ Day 1 — First correction: bullets → table0.50
🗓️ Day 3 — Second correction: bullets → table again0.65
🗓️ Day 7 — Trend detected, multiplier 1.15x0.75
🗓️ Day 14 — "Yes, tables are perfect!" feedback0.88

Result: System learns user strongly prefers tables. Next API design session auto-uses table format.

🔗 Option C — Diminishing Confidence Loading

Load primary preference (confidence: 0.88)
  ↓ follow association (strength: 0.80) → confidence × 0.80 = 0.70
    ↓ follow association (strength: 0.60) → confidence × 0.60 = 0.42
      STOP — below threshold (0.45)

Max depth: 3 levels
Threshold: 0.45
Smart discovery through association chains

🤖 Agent Integration
How to wire the engine into an AI agent in minutes

⚡ Minimal Integration

from scripts.agent_hook import AgentPreferenceHook

hook = AgentPreferenceHook()

# Get preferences for context (returns JSON)
prefs = hook.get_preferences_for_context(["python", "api_design"])

# Use preferences in your agent response...

# Report when user corrects you
hook.report_correction(
    task="api_design",
    context_tags=["python"],
    agent_proposed="bullets",
    user_corrected_to="table",
    user_message="Perfect, tables work better here!"
)

🔌 Middleware — Auto-inject into Prompts

from scripts.agent_hook import AgentPreferenceMiddleware

middleware = AgentPreferenceMiddleware()

# Preferences injected directly into the prompt
enhanced_prompt = middleware.inject_preferences(
    base_prompt="Help me design a Python API",
    context_tags=["python", "fastapi"]
)

# enhanced_prompt now includes:
# 🎯 ASSISTANT PREFERENCES (learned from user behavior):
# PRIMARY: communication.output_format.table
#   Confidence: 88%
# RELATED:
#   • coding.data_structure_clarity (Confidence: 70%)

⌨️ CLI Reference
Full command reference for adaptive-cli
Create
# Create a preference
adaptive-cli pref create --name bullets --path communication.output_format.bullets --type variant

# Create an association
adaptive-cli assoc create \
  --from-id communication.output_format.table \
  --to-id coding.data_structure_clarity \
  --strength-forward 0.95 \
  --strength-backward 0.70

# Create a context
adaptive-cli context create --name "FastAPI Project" --scope project
Learn
# Record a correction
adaptive-cli signal correction \
  --task api_design \
  --context python fastapi \
  --proposed communication.output_format.bullets \
  --corrected communication.output_format.table \
  --message "Yes, tables are perfect for API endpoints!"

# Record feedback
adaptive-cli signal feedback \
  --task code_explanation \
  --context python \
  --preferences communication.output_format.bullets communication.depth.detailed \
  --response "Crystal clear! Exactly the level of detail I needed." \
  --satisfaction 0.95
View
# List and inspect
adaptive-cli pref list
adaptive-cli pref show <id>
adaptive-cli assoc show <id>
adaptive-cli context show <id>

# Load preferences for context
adaptive-cli load --context python api_design

# Generate JSON for agent
adaptive-cli agent-context --context python fastapi
Maintain
adaptive-cli recalculate   # Recalculate all strengths
adaptive-cli decay         # Apply time decay (2%/day)
adaptive-cli stats         # Show statistics
adaptive-cli reset         # Reset everything
🚀 Quickstart
Get up and running in under 5 minutes

1. Install

cd adaptive-preference-engine
chmod +x scripts/cli.py     # optional: make CLI executable

2. Run the Demo

python demo.py

Demo shows: creating preferences → associations → contexts → corrections → learned preferences → agent JSON output

3. Create Your First Preference

python scripts/cli.py pref create \
  --name table \
  --path communication.output_format.table \
  --type variant

4. Record a Correction

python scripts/cli.py signal correction \
  --task api_design \
  --context python \
  --proposed communication.output_format.bullets \
  --corrected communication.output_format.table \
  --message "Tables work much better here!"

5. Check What the Engine Learned

python scripts/cli.py agent-context --context python api_design
